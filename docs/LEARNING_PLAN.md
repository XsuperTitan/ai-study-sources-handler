# 学习计划 / PLAN

本文记录当前 `学习计划 / PLAN` 功能的产品边界、交互流程、API、数据模型和验证方式。

## 当前范围

PLAN 仍只维护一个活跃学习计划。用户从首页资料卡片集把资料加入计划草稿，点击生成后才调用 LLM；生成后的计划可进入 `/plan` 工作台查看排期、记录学习、完成步骤，并按需请求 AI 重排。

已实现能力：

- 首页保留轻量 PLAN 摘要，展示总进度、今日任务提示、阶段方块和进入工作台入口。
- 首页右下角提供可自由拖动的悬浮投放区，左侧 `PLAN` 竖条作为拖动把手。
- 资料卡片支持原生 HTML Drag and Drop，也支持卡片右上角“加入学习计划”按钮。
- 悬浮投放区展示已选资料，支持删除不想纳入计划的资料，并提供生成/重新生成按钮。
- PLAN 摘要和工作台提供一键重置，确认后清空当前计划草稿、步骤、学习记录、版本和重排草稿，但不删除资料卡片。
- 待规划资料按加入顺序保存，重复加入会去重。
- 只允许 `READY` 和 `PARTIALLY_READY` 资料包参与生成；处理中资料可显示，但生成时会提示不可用。
- `/plan` 工作台展示圆环总进度、方块阶段进度、本周/今日任务、步骤时间线、关联资料、学习记录和 AI 重排面板。
- 总进度按 `已完成步骤数 / 总步骤数` 计算；学习分钟用于统计和重排上下文，不直接改变完成率。

## 用户流程

```mermaid
flowchart LR
    A[首页资料卡片] -->|拖入或点击加入| B[悬浮 PLAN 投放区]
    B --> C[待规划资料清单]
    C -->|删除| C
    C -->|生成/重新生成| D[DeepSeek JSON 计划生成]
    D --> E[当前学习计划]
    E --> F[/plan 工作台]
    F --> G[记录学习分钟与笔记]
    F --> H[完成/取消完成步骤]
    F --> I[AI 重排建议]
    I -->|用户确认| E
```

### 首页

- 悬浮投放区初始固定在右下角。
- 用户可拖动投放区左侧 `PLAN` 把手，位置会被限制在当前视口内。
- 拖拽资料卡片进入投放区时，资料卡片半透明，投放区高亮。
- 删除某个已选资料只会更新待规划清单，不会删除资料包本身。
- 生成失败时保留已选资料和现有计划，并显示可重试错误。

### PLAN 工作台

`/plan` 是完整学习体验页面：

- 左侧展示今日任务和本周节奏。
- 中间展示方块阶段进度和步骤时间线。
- 右侧展示所选步骤的排期、反思、学习记录、关联资料和 AI 重排入口。
- 完成步骤后即时刷新圆环总进度和方块状态。
- 记录学习分钟不会强制完成步骤。
- 重排建议先预览摘要，用户点击“应用建议”后才覆盖未完成步骤并递增版本。

## API

保留和扩展的接口：

| 方法 | 路径 | 用途 |
|---|---|---|
| `GET` | `/api/v1/learning/plan` | 返回当前计划、待规划资料、步骤、今日任务、周摘要和进度 |
| `DELETE` | `/api/v1/learning/plan` | 重置当前学习计划，清空草稿、步骤、记录、版本和重排建议 |
| `PUT` | `/api/v1/learning/plan/packages` | 保存当前计划引用的资料包顺序 |
| `POST` | `/api/v1/learning/plan/generate` | 基于当前资料包调用 `AiProviders.deepSeekJson` 生成计划 |
| `PATCH` | `/api/v1/learning/plan/steps/{stepId}` | 标记步骤完成或未完成 |
| `POST` | `/api/v1/learning/plan/steps/{stepId}/sessions` | 记录一次学习分钟、笔记和学习时间 |
| `PATCH` | `/api/v1/learning/plan/steps/{stepId}/schedule` | 调整步骤日期、预计分钟和反思 |
| `POST` | `/api/v1/learning/plan/replan` | 基于完成情况、记录和反馈生成重排建议 |
| `POST` | `/api/v1/learning/plan/replan/{proposalId}/apply` | 应用已确认的重排建议 |

关键错误码：

| 错误码 | 场景 |
|---|---|
| `LEARNING_PLAN_PACKAGE_NOT_FOUND` | 保存的资料包 ID 不存在 |
| `LEARNING_PLAN_PACKAGE_NOT_READY` | 资料包仍在处理或不可用于生成 |
| `LEARNING_PLAN_EMPTY_PACKAGES` | 未选择资料包却请求生成 |
| `LEARNING_PLAN_RESPONSE_INVALID` | LLM JSON 为空、格式不合法或缺少必要字段 |
| `LEARNING_PLAN_STEP_NOT_FOUND` | 步骤不存在或不属于当前计划 |
| `LEARNING_PLAN_PROPOSAL_NOT_FOUND` | 重排建议不存在或已被应用/清理 |

## 数据表

Flyway 迁移：

- `V2__create_learning_plan_tables.sql`
- `V3__extend_learning_plan_workspace.sql`

核心表：

| 表 | 用途 |
|---|---|
| `learning_plan` | 单活跃计划主表，保存标题、概览、总分钟、版本和生成时间 |
| `learning_plan_package` | 计划引用资料包，保存标题和关键词快照，避免资料后续删除或改名导致展示断裂 |
| `learning_plan_step` | 计划步骤，保存排期、预计分钟、实际分钟、阶段标签、完成时间、反思和关联资料 |
| `learning_plan_study_session` | 每次学习记录，保存分钟数、笔记和学习时间 |
| `learning_plan_version` | 生成/重排版本记录，保存 LLM 输入摘要和输出快照 |
| `learning_plan_replan_proposal` | 待确认 AI 重排建议 |

## LLM 约束

生成计划时输入每个资料包的：

- 标题
- 类型
- 关键词
- 学习指南摘要
- 复习建议

输出必须是可解析 JSON，并包含计划标题、概览、预计分钟、本周摘要、步骤列表、步骤排期、预计分钟、阶段标签和关联资料。后端会校验必要字段，解析失败返回可重试错误，不清空草稿。

重排时输入包含：

- 当前计划版本
- 已完成步骤
- 未完成步骤
- 学习记录
- 用户反馈
- 当前资料包快照

重排结果先写入 proposal 表，只有用户确认后才更新当前步骤并写入新版本。

## 前端实现要点

- 不新增拖拽依赖，使用原生 Drag and Drop 和 React 状态管理。
- 悬浮投放区的移动使用 Pointer Events，只在 `PLAN` 把手上启动拖动，避免影响删除和生成按钮点击。
- 首页和 `/plan` 共用 `learning-plan` 查询缓存；生成、完成步骤、记录学习和应用重排后直接刷新缓存。
- 进度视觉采用“圆环总进度 + 方块阶段进度”。
- 资料卡片的“加入学习计划”按钮用于键盘和无拖拽用户。

## 验证

已覆盖的测试：

- 后端单元测试：保存顺序、去重、只允许可生成状态、LLM JSON 失败、步骤进度、学习记录、重排建议和版本递增。
- 后端控制器测试：计划查询、保存资料、生成、完成步骤、记录学习、调整排期、重排和应用建议。
- 前端测试：首页摘要、悬浮投放区拖入、点击加入、删除资料、生成失败保留草稿、完成步骤刷新进度、工作台记录学习和应用重排。
- E2E：从首页拖入资料包，生成计划，进入工作台，完成一步并验证进度变化。

常用验证命令：

```powershell
cd backend
.\mvnw.cmd test

cd ..\frontend
npm run test
npm run build
```

## 当前边界

- v2 仍只支持一个当前活跃计划。
- AI 生成和重排只在用户点击时触发。
- 重排只覆盖未完成步骤；已完成步骤用于上下文和版本记录。
- 学习分钟用于统计和 AI 重排，不直接改变总进度。
- 不支持多个计划文件夹、复杂排序、日历提醒或自动复习通知。
